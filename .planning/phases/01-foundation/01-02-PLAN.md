    - isGitRepo returns false for non-repo directory
    - addFiles stages specified files in the repo
    - commitFiles creates a commit with the specified message
    - writeGitattributes creates .gitattributes with LF enforcement content
  </behavior>
  <action>
Write tests first (RED), then implement (GREEN).

1. Create `tests/git/repo.test.ts`:
   - Use real temp directories (fs.mkdtemp in os.tmpdir())
   - Test each git operation independently
   - Clean up temp dirs in afterEach

2. Create `src/git/repo.ts`:
   - Import simpleGit from 'simple-git'
   - Export `async function initRepo(repoPath: string): Promise<void>`
     - Verify directory exists, throw if not
     - Call `simpleGit(repoPath).init()`
   - Export `async function isGitRepo(dirPath: string): Promise<boolean>`
     - Try `simpleGit(dirPath).checkIsRepo()`, return result
     - Catch errors, return false
   - Export `async function addFiles(repoPath: string, files: string[]): Promise<void>`
     - Call `simpleGit(repoPath).add(files)` -- always use explicit file paths, never `git add .`
   - Export `async function commitFiles(repoPath: string, message: string): Promise<void>`
     - Call `simpleGit(repoPath).commit(message)`
   - Export `async function writeGitattributes(repoPath: string): Promise<void>`
     - Write `.gitattributes` file with content:
       ```
       * text=auto eol=lf
       *.json text eol=lf
       *.md text eol=lf
       *.js text eol=lf
       *.sh text eol=lf
       ```
     - Use `fs.writeFile(path.join(repoPath, '.gitattributes'), content)`

3. Create `src/cli/index.ts`:
   - Import { Command } from 'commander'
   - Create program: name "claude-sync", description "Git-backed sync for ~/.claude", version from package.json (import with assert)
   - Import and call `registerInitCommand(program)`
   - Export program
   - At bottom: `program.parse()` (only when run directly, not imported)

4. Create `src/cli/commands/init.ts`:
   - Export `function registerInitCommand(program: Command): void`
   - Register "init" subcommand with:
     - Option `--force` to re-initialize
     - Option `--repo-path <path>` with default from getSyncRepoDir()
     - Action handler (async) that orchestrates the full init flow:
       a. Resolve claudeDir (getClaudeDir()) and syncRepoDir (getSyncRepoDir or --repo-path)
       b. Check if claudeDir exists (error if not: "No ~/.claude directory found")
       c. Check if syncRepoDir already exists and is a git repo (error unless --force: "Sync repo already exists at {path}. Use --force to re-initialize.")
       d. Create syncRepoDir with `fs.mkdir(syncRepoDir, { recursive: true })`
       e. Call `initRepo(syncRepoDir)`
       f. Call `writeGitattributes(syncRepoDir)`
       g. Call `addFiles(syncRepoDir, ['.gitattributes'])`
       h. Call `commitFiles(syncRepoDir, 'chore: initialize sync repo with line ending config')`
       i. Call `scanDirectory(claudeDir)` to get list of files to sync
       j. For each file in the list:
          - Read from claudeDir
          - If filename is "settings.json": apply `rewritePathsForRepo(content, getHomeDir())`
          - Ensure parent directory exists in syncRepoDir with `fs.mkdir`
          - Write to syncRepoDir
       k. Call `addFiles(syncRepoDir, copiedFiles)` with the list of all copied relative paths
       l. Call `commitFiles(syncRepoDir, 'feat: initial sync of claude config')`
       m. Print summary using picocolors: files synced count, files excluded, sync repo location
   - Use picocolors for colored output (green for success, yellow for warnings, red for errors)

5. Update `src/index.ts` to also re-export from git/repo.

6. Run tests to confirm GREEN.
